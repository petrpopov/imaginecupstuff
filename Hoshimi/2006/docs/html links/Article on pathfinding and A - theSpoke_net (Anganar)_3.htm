Here is the code of the MoveTo(Point[]) function using Reflector :

public unsafe bool MoveTo(Point[] path)
{
      if (base.State != NanoBotState.WaitingOrders)
      {
            return false;
      }
      if ((path == null) || (path.Length == 0))
      {
            base.setState(NanoBotState.WaitingOrders);
            return false;
      }
      base.setPointInfo(path[path.Length - 1]);
      if (this.MovePoints == null)
      {
            this.MovePoints = new Stack<Point>();
      }
      this.MovePoints.Clear();
      for (int num1 = path.Length - 1; num1 >= 0; num1--)
      {
            this.MovePoints.Push(*((Point*) &(path[num1])));
      }
      base.setState(NanoBotState.Moving);
      return true;
}

And as you can see, it doesn't use the internal PathFinder !

The connection between two points need to be horizontal/vertical only !! and it's not in
this method you'll find this but somewhere where the move is done :)

If you use the Internal Pathfinder it's only by using the function : moveTo(Point DESTINATION)
(the location, is your current location) ... There is another way, 
but a little more complicated :)

About the API (I'm speaking from my last year' experience) : there were changes only when 
there were bugs but they were valid only for the next round (I mean, if a bug was discover 
in the round 1, the change was done for the round 2 in the SDK for the round 2)
