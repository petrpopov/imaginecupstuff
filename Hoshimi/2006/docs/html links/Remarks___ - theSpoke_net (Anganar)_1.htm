Yeah, I also would like to be able to debug my code using VS Express by running my dll in the 2Dviewer - well actually I am. I searched the config files and found the relevant string in "PlayerSampleMission1CSharp.csproj.user" (for C# coders, but who uses VB anyway). Just replace two times the given string "C:\Documents and Settings\t-bgauth\Mes documents\Hoshimi\src10" in the StartProgram property with your individual "X:\XXX\..\Microsoft Image Cup 2006" and voila using the "start debugging" function the Viewer starts and you can debug your dll.

If anyone can tell where to set this the vanilla way in the IDE I would be happy to learn about it ;).
